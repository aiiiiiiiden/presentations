## Claude Code 헤드리스 모드란?

- **인터랙티브 모드** (`claude`): TUI(대화 화면)를 띄우고 다회 턴 주고받는 일반 사용 형태.
- **헤드리스 모드** (`claude -p "..."`): TUI 없이 _단일 명령 → 단일 응답 → 종료_. CI·스크립트·자동화에 주로 사용.
- **이 데모에서의 용도:** *맥락 없는 순수 LLM 호출*을 시각화. 다만 헤드리스만으로는 Plugin·내장 Tools·MCP가 여전히 자동으로 붙어 돌아간다. 그래서 **디렉토리에 `.claude/settings.json`을 두고 명시적으로 차단** — **Model + LLM + Prompt 외엔 아무것도 작동하지 않는 맨몸 상태**를 만든다.

## 순수 LLM 호출용 settings.json

**파일 위치:** `~/presentations/20260514/demo/01-no-context/.claude/settings.json`

```json
{
  "permissions": {
    "deny": [
      "Bash",
      "Read",
      "Write",
      "Edit",
      "Grep",
      "Glob",
      "WebSearch",
      "WebFetch",
      "Task",
      "TodoWrite",
      "NotebookEdit"
    ]
  },
  "skillsMode": "off",
  "enabledPlugins": {},
  "enableAllProjectMcpServers": false,
  "disabledMcpjsonServers": ["*"]
}
```

**차단 매핑:**

| 항목                                                                                       | 차단 키                                                                                |
| ------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------- |
| 내장 Tools (Bash·Read·Write·Edit·Grep·Glob·WebSearch·WebFetch·Task·TodoWrite·NotebookEdit) | `permissions.deny`                                                                     |
| MCP 도구 호출 (user·project 모두)                                                          | `permissions.deny: ["mcp__*"]` 와일드카드                                              |
| Skills 자동 호출                                                                           | `skillsMode: "off"`                                                                    |
| Plugins                                                                                    | `enabledPlugins: {}`                                                                   |
| Project MCP 서버                                                                           | `enableAllProjectMcpServers: false` + `disabledMcpjsonServers: ["*"]` + 빈 `.mcp.json` |
| CLAUDE.md                                                                                  | 디렉토리에 없음 (자동)                                                                 |

## 시연 절차

```bash
cd ~/presentations/20260514/demo/01-no-context
claude -p \
  --setting-sources project \
  "SK하이닉스의 내일 주가를 상승, 하락, 횡보 확률을 예측해줘"
```

**플래그 의미:**

- `-p` (print) — **헤드리스 모드**. 인터랙티브 세션 없이 stdout으로 답변 출력 후 종료.
- `--setting-sources project` — **user-level `~/.claude/settings.json` 로드 차단**. 이 디렉토리의 `.claude/settings.json`만 적용돼 *맨몸 환경*이 보장됨. (user settings에 있는 plugin·skill·hook 등이 새어 들어오지 못함)

> 헤드리스(`-p`) + `--setting-sources project` + 디렉토리 settings.json 조합이 딱 **Model + LLM + Prompt만 작동**하는 가장 깨끗한 셋업.
>
> 인증(OAuth/keychain)은 그대로 유지됨. `--bare`처럼 인증을 깨지 않음.

## 발표 스크립트

> 자, 첫 번째 단계입니다.
>
> 시연에 들어가기 전, *Claude Code의 헤드리스 모드*를 30초만 짚겠습니다.  
> 평소 쓰는 Claude Code는 터미널에서 사용자 입력을 **인터랙티브 모드로 처리**합니다.  
> 그런데 claude를 실행할 때 `-p` 플래그를 붙이고 입력을 함께 넘기면, *한 줄 명령을 받아 답변한 뒤 종료*되는 헤드리스 모드로 떨어집니다.
>
> 헤드리스로 실행돼도 제 개발 환경의 전역 설정이 살아 있으면 _Plugin, 내장 Tools, Skills, MCP_ 가 그대로 붙어 돌아갑니다.  
> **4장에서 다룬 7개 개념 중 *첫 번째 묶음인 Model, LLM, Prompt*만 남아 있는 상태**를 보려면, claude가 실행되는 경로에 `.claude/settings.json` 파일을 두고 도구 실행 권한을 빼고, skill·plugin·mcp 실행까지 막아 둡니다.
>
> _이 상태에서 `-p`로 헤드리스 실행하면_ **Model, LLM, Prompt**만 남은 가장 깨끗한 환경이 됩니다.  
> 순수 model, llm, prompt가 어떻게 동작하는지 보려고 미리 만들어 둔 디렉토리로 옮겨 가서 "내일 예상되는 SK하이닉스의 주가를 상승/하락/횡보 확률로 예측해줘."라고 던집니다.
>
> ```
> cd ~/presentations/20260514/demo/01-no-context
> claude -p \
>   --setting-sources project \
>   "내일 예상되는 SK하이닉스의 주가를 상승/하락/횡보 확률로 예측해줘"
> ```
>
> *`--setting-sources project`* 플래그는 _user-level 설정(`~/.claude/settings.json`) 로드를 끊어_, 이 디렉토리의 settings.json만 먹히게 합니다. user 환경에 깔린 plugin·skill·hook이 새어 들어오지 못하게 잡아주는 안전핀입니다.

> 어떤 응답이 내려왔나요?
>
> ```
> 내일(2026-05-12) SK하이닉스 주가 예측
>
> | 시나리오 | 확률 |
> |---------|------|
> | 상승 | 약 42% |
> | 하락 | 약 38% |
> | 횡보(±0.5% 이내) | 약 20% |
>
> **근거 (LLM 일반 지식 기반):**
> - HBM3E/HBM4 수요가 AI 가속기 사이클과 함께 견조한 흐름
> - 메모리 업황은 2025년 하반기부터 회복 국면 진입한 것으로 알려짐
> - 한국 대형주의 일간 변동성 통계상 ±1~2% 구간이 가장 빈번
>
> **중요한 한계 — 솔직하게 말씀드립니다:**
>
> 이 예측은 사실상 **추측에 가깝습니다.** 그 이유는:
>
> 1. **저는 실시간 데이터에 접근할 수 없습니다.** 오늘 종가, 외국인/기관 수급, 옵션 만기, 환율, 미국 증시 마감, 엔비디아 실적, OPEC 동향 — 무엇 하나 모릅니다.
> 2. **제 지식은 2026년 1월에서 멈춰 있습니다.** 지난 4개월간 무슨 일이 있었는지 알지 못합니다.
> 3. **단일 종목의 익일 방향성은 학술적으로 예측 불가능에 가깝습니다.** 위 확률은 본질적으로 "동전 던지기에 약간의 편향을 더한 값"에 불과합니다.
>
> 위 숫자는 제가 가진 일반적인 시장 직관을 정량화한 것이지, 신뢰할 만한 예측이 아닙니다. 실제 투자 판단에 사용하지 마십시오.
> ```
>
> 그럴싸한 근거를 붙여 상승 42%, 하락 38%, 횡보 20%라고 찍어내긴 했는데,  
> 마지막 학습이 2026년 1월이라 실시간 데이터에 손을 댈 수 없었고, 편향 섞인 예측이라기보다 추측에 가깝다고 자기 입으로 인정합니다.
>
> LLM이 학습 데이터에 기대 확률로 응답을 뽑긴 합니다.  
> 그래도 같은 모델이면 내일 돌리든 다음 주에 돌리든 학습 데이터가 그대로라 비슷한 답이 나옵니다.  
> 자연어 입력을 알아듣는 솜씨는 뛰어나도, 결국 검색엔진과 별반 다르지 않습니다.  
> 이 결과로 주식 계좌에 남은 5천만원을 굴리기엔 모자랍니다.
>
> 그럼 뭐가 부족해서 이런 결과가 나왔을까요?  
> 답은 **맥락**입니다.  
> AI에게서 좋은 답을 받으려면 — *질문에 맥락을 충분히 같이 얹어 주는 것*이 핵심입니다.

## 발표자 노트

- **연출 의도:** 이 단계는 *맨몸 LLM의 한계*를 청중이 _직접 읽고_ 체감하게 만드는 자리. 응답이 _"실시간 데이터 접근 불가, 학습은 2026년 1월에 멈춤, 익일 방향성 예측 불가"_ 라며 **AI 스스로 한계를 털어놓는다** — 이 대목을 청중이 스스로 읽도록 호흡을 두면 _"AI도 자기 한계를 안다"_ 와 _"그럼 어떻게 채우지?"_ 두 인식이 함께 떠오른다. 6-2의 변화가 _대비로_ 살아난다.
- `**--bare` 대신 settings.json을 고른 이유:\*\*
  - `--bare` 플래그는 *OAuth/keychain 인증까지 끊어버려* 발표 중 `Not logged in` 에러가 터질 위험이 크다 (GitHub #39069, #51047).
  - 디렉토리 + settings.json 방식은 *인증은 살려둔 채* Plugin·Tools·Skills·MCP만 차단해 — 발표 안정성이 올라간다.
  - 청중 입장에서도 _"이 디렉토리는 맨몸 환경"_ 이라는 메시지가 **눈에 보이는 파일 한 장**으로 닿아 한눈에 들어온다.
- **settings.json 시각화 시점:** settings.json 전문을 슬라이드에 띄우거나 `cat .claude/settings.json`으로 한 번 보여주는 자리가 있으면 좋다. 다만 비개발자가 JSON을 길게 들여다보면 눈이 풀리니 — _핵심 키 4개만 짚는다_: `permissions.deny`, `skillsMode: "off"`, `enabledPlugins: {}`, `disabledMcpjsonServers: ["*"]`.
- **호흡 (4분 분배):**
  - ① 도입 + 헤드리스 모드 소개 — 약 33초
  - ② settings.json 차단 설명 (7개 개념 회수 라인 포함) — 약 70초
  - ③ 명령 입력 + `--setting-sources project` 플래그 부연 — 약 27초
  - ④ 응답 dead time (LLM 응답 생성 + 청중 침묵 + 한계 단락 짚기) — 약 25초
  - ⑤ 응답 해석 + "맥락이 부족하다" 진단 + 다리 — 약 77초
  - + 슬라이드 전환·시선 — 약 10초
  - **총 ~242초 ≈ 4:02**. 4분 안정 페이스.
- **응답 처리 원칙:**
  - 응답은 _예측 표 + 한계 설명 세 가지_ 꼴. 발표자가 한 줄씩 읽어주면 시간 잡아먹고 동력만 빠진다.
  - _"보세요. 어떤 답이 내려왔나요?"_ 한 마디 던지고 **5~7초 침묵** — 청중이 화면을 직접 읽도록 둔다.
  - 발표자는 _세 번째 단락(한계 설명)만 짚는다_: "실시간 데이터 접근 불가, 학습은 2026년 1월에 멈춤, 익일 방향성 예측 불가." 청중에게 보이는 부분이 있고 발표자는 안 보이는 부분을 채우는 _역할 분담_.
- **핵심 회수 한 줄 (반드시 박을 것):** _"AI에게서 좋은 답을 받으려면 — 질문에 맥락을 충분히 같이 얹어 주는 것이 핵심입니다."_ 이 한 줄이 6-1을 *단순 데모*에서 *청중이 자기 일로 가져갈 가르침*으로 끌어올린다. 천천히, 단어 사이 호흡.
- **금기 표현:**
  - "사실 이건 별거 아닙니다" — 비하 금지. 사실로만.
  - "이 답이 틀렸어요" — 틀린 게 아니라 *맥락이 빠진 것*이다.
  - "AI는 결국 검색엔진 수준입니다" — 본문의 *"검색엔진과 별반 다르지 않습니다"*는 *맥락 없는 상태*에서만 통하는 표현. 일반화 금지.
- **사전 준비:**
  - `~/presentations/20260514/demo/01-no-context/` 디렉토리에 `.claude/settings.json`, `.mcp.json` 사전 배치 (이미 생성됨).
  - 발표 전 같은 명령으로 리허설 — 응답이 표로 떨어지는지, 평문 단락으로 떨어지는지 확인한다. **표가 안 나오면 응답 출력 슬라이드를 백업 캡처로 준비.**
  - 터미널 글꼴 크기 충분히 키워두기 (헤드리스 출력은 일반 텍스트라 줄바꿈 가독성이 관건).
  - **User-level MCP 사전 점검:** `~/.claude.json`에 MCP 서버가 등록돼 있으면 *서버 프로세스*가 뜬다. settings.json의 `enableAllProjectMcpServers: false` + `disabledMcpjsonServers: ["*"]`로 *project MCP*는 막히지만 user-level은 별개다. 더 엄격하게 보여주려면 발표 직전 user MCP를 잠시 꺼두거나 `--strict-mcp-config --mcp-config '{}'` 옵션을 더한다.
- **모드 전환 안내:** 6-2부터는 *다른 디렉토리(또는 settings.json 없는 디렉토리)*에서 인터랙티브 `claude`로 띄운다. _"다음 단계부터는 인터랙티브 모드로 띄워서"_ 한 줄로 자연스럽게 다리 놓는다.
- **다리 문장:** _"답은 맥락입니다. AI에게서 좋은 답을 받으려면 — 질문에 맥락을 충분히 같이 얹어 주는 것이 핵심입니다. 다음 단계부터는 맥락을 차곡차곡 넣어가며 결과를 끌어올려 보겠습니다."_ → 슬라이드 — "맥락 추가" 한 단어 → 6-2 시작.
