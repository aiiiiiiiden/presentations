**시간:** 22:30–25:30 (3분)

> ⚠ 시작 시각은 01~06-1 시프트(+3.5분)에 따라 19:00 → 22:30으로 조정. 분량은 3분 유지하되 *dead time 변수에 주의* — PDF 30쪽 추론이 40초면 버퍼 13초, 60초면 7초 초과. 리허설로 dead time 실측 후 ⑥ 편향 경고 단락에서 8초 정도 압축 여부 결정.

## 02-context의 settings.json

01-no-context와 거의 같다. 차이는 하나, `**Read` 도구를 `permissions.allow`에 명시적으로 추가**해 PDF를 읽을 수 있게 했다.

```json
{
  "permissions": {
    "allow": ["Read"],
    "deny": [
      "Bash",
      "Write",
      "Edit",
      "Grep",
      "Glob",
      "WebSearch",
      "WebFetch",
      "Task",
      "TodoWrite",
      "NotebookEdit"
    ]
  },
  "skillsMode": "off",
  "enabledPlugins": {},
  "enableAllProjectMcpServers": false,
  "disabledMcpjsonServers": ["*"]
}
```

> **왜 `allow`에 명시해야 할까?**
> 헤드리스 모드(`-p`)는 사용자에게 권한 승인을 묻지 못한다. `deny`에 없더라도 *명시적으로 `allow`되지 않은 도구*는 자동 거부된다. PDF를 읽으려면 `Read`를 `allow`에 넣어두는 게 필수. (인터랙티브 모드에선 사용자가 그때그때 승인할 수 있어 차이가 덜 두드러진다.)
>
> Skills·Plugins·MCP는 세 겹으로 막아뒀다:
>
> - `permissions.deny`에 도구 차단 + `mcp__`* 와일드카드(필요 시) — MCP 도구 호출 자체를 막는다.
> - `enableAllProjectMcpServers: false` + `disabledMcpjsonServers: ["*"]` — project MCP 자동 활성화도 꺼두고 전부 비활성.
> - `.mcp.json`은 빈 객체.
>
> 추가로 푼 권한은 "파일 한 장 읽기"(`Read`)뿐이다. 외부 검색이나 웹 호출은 그대로 막혀 있다.

```bash
cd ~/presentations/20260514/demo/02-context
claude -p \                                  
  --setting-sources project \
  "SK하이닉스의 종목 리포트인 **20260511_[반도 체] SK하이닉스 (000660_매수).pdf** 파일을 참고해서 SK하이닉스의 주가를 상승, 하락, 횡보 확률로 예측해줘"
```

**플래그 의미:**

- `-p` — 헤드리스 모드 (한 줄 명령에 한 줄 응답).
- `--setting-sources project` — user-level `~/.claude/settings.json`을 아예 안 읽는다. project settings만 적용.

> 답이 어떻게 달라졌나요?
>
> 6-1에서는 *"추측에 가깝다"*였는데, 이번엔 PDF에 적힌 애널리스트 목표가, HBM 양산 일정, 매수 의견을 직접 인용하면서 근거를 댑니다. 학습 데이터에 없던 *2026년 5월의 정보*가 PDF 한 장으로 들어온 겁니다.
>
> *리포트라는 맥락, 곧 Context를 주고 그걸 참조하라고 지시했더니 결과가 어떻게 달라졌나요?*
> 리포트 덕에 SK하이닉스 주가 예측의 신뢰도는 올라갔지만, 거꾸로 리포트에 끌려가는 편향이 생길 수 있습니다. 매수 리포트였으면 상승 확률이, 매도 리포트였으면 하락 확률이 높게 나왔을 테니까요.

- **연출 의도:** 6-1과의 대비가 이 단계의 핵심이다. "명령도 프롬프트도 그대로, PDF 한 장만 더했다"는 사실을 반복해서 깔아야 청중이 "맥락 = 외부 자료 참조"라는 정의를 스스로 잡는다.
- **6-1 응답을 화면 한쪽에 띄울 것:** 가능하면 슬라이드를 좌우로 갈라 6-1 응답(추측 자백)과 6-2 응답(PDF 근거 인용)을 나란히 띄운다. 청중이 눈으로 비교하면 발표자가 입으로 설명할 분량이 줄어든다.
- **호흡 (3분 분배):**
  - ① 6-1 회수 + 한계 자백 인용 — 약 20초
  - ② 변화 안내 ("리포트 추가") + 명령 입력·타이핑 — 약 15초
  - ③ PDF 읽기 + 응답 생성 dead time — 약 40~60초 (변수가 가장 큰 구간)
  - ④ "답이 어떻게 달라졌나요?" + 청중 반응을 기다리는 침묵 — 약 10초
  - ⑤ 비교 진단 (PDF 인용 부분 짚기) — 약 25초
  - ⑥ 편향 경고 + 6-3 다리 — 약 25초
- **Dead time 운용:** ③ 구간 40~60초가 발표 흐름에서 가장 빈 시간이다. 침묵하지 말고 *"Claude가 지금 PDF를 한 페이지씩 읽고 있습니다. 30쪽짜리 리포트라 잠깐 걸립니다"* 식으로 지금 화면 뒤에서 무슨 일이 벌어지는지 청중에게 중계한다. 또는 PDF 표지를 슬라이드에 띄워 어떤 정보가 들어가는지 미리 보여준다. 리허설로 응답 시간을 재두면 dead time 길이에 맞춰 대본을 조정한다.
- **핵심 회수 한 줄 (반드시 박을 것):** *"프롬프트는 그대로. 명령도 그대로. 바뀐 건 하나, 디렉토리에 맥락 파일 한 장."* 이 세 문장을 같은 리듬으로 반복하면 청중 머리에 메시지가 박힌다.
- **맥락 정의의 진화:** 4장에서 *"Prompt에 맥락을 추가"*로 정의했고, 6-1에서 *"맥락이 비어있다"*로 자각, 6-2에서 *"맥락 = 외부 자료를 LLM에게 참조하게 해주는 것"*으로 넓어진다. 이 진화 곡선을 1장에서 단숨에 짚어준다.
- **금기 표현:**
  - "AI가 PDF를 이해했어요" — 의인화 금지. PDF를 읽고 그 내용을 답변에 반영했다고 정확히 말한다.
  - "이제 진짜 분석가네요" — 과장 금지. 여전히 LLM이 PDF를 인용하는 수준이다. 진짜 분석은 6-7 sub-agent에서 다룬다.
  - "PDF만 넣으면 다 해결됩니다" — 단순화 금지. 이렇게 말해버리면 뒤에 나올 페르소나·Skill·MCP가 왜 필요한지 흐름이 끊긴다.
- **사전 준비:**
  - `~/presentations/20260514/demo/02-context/` 디렉토리에 `.claude/settings.json`, `.mcp.json`, SK하이닉스 PDF를 미리 깔아둔다 (이미 생성됨).
  - settings.json의 `permissions.deny`에서 Read만 빠져 있는지 확인 — 나머지 차단은 6-1과 똑같고 `mcp__`* 와일드카드 deny가 추가됐다.
  - CLI 플래그 두 개 (`--setting-sources project`, `--strict-mcp-config --mcp-config '{}'`)로 user 환경의 영향을 원천 차단한다. settings.json만 가지고는 user-level MCP 서버를 로드하려는 시도까지는 못 막기 때문에 (deny는 호출 단계에서만 막는다), 이 플래그로 아예 로드 자체를 안 시킨다.
  - 발표 전에 같은 명령으로 리허설을 돌려 Claude가 PDF를 알아서 읽는지 확인한다. 안 읽으면 프롬프트 끝에 *"이 디렉토리의 PDF를 참고해서"* 한 줄을 덧붙이는 백업 명령을 준비해둔다.
  - 응답이 표로 나올지 평문으로 나올지 모르니 캡처본을 양쪽 다 챙긴다.
- **다음 단계 다리:** "다음 단계에서는, 이 맥락 위에 *전문가의 사고 절차*까지 얹어보겠습니다." → 6-3 페르소나로 진입.
