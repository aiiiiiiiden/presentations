## 시연 절차

### 1) 메타-스킬 파일 (이미 저장돼 있음)

`~/presentations/20260514/demo/05-meta-skill/.claude/skills/sector-analyst-creator/SKILL.md`

```markdown
---
name: sector-analyst-creator
description: 새 섹터(반도체·디스플레이·배터리·바이오 등) 전문 애널리스트 페르소나를 자동 생성. 트리거 — "X 섹터 전문가 만들어줘", "Y 애널리스트 스킬 추가"...
---

# 섹터 애널리스트 생성기

작업 절차:
1) 입력 파싱 — "디스플레이" → 슬러그 `display`
2) WebFetch로 섹터 리서치 (위키·협회·리서치 페이지)
3) `.claude/skills/<sector>-analyst/SKILL.md` 자동 작성 (분석 원칙 5개·출력 형식·금기)
4) 완료 보고 (경로·요약·트리거 키워드)
```

### 2) settings.json 차이 (04 → 05)

04-skill 대비 `WebSearch`·`WebFetch`·`Write` 세 도구를 `permissions.allow`에 추가.

```json
{
  "permissions": {
    "allow": ["Read", "Skill", "WebFetch", "WebSearch", "Write"],
    "deny": ["Bash", "Edit", "Grep", "Glob", "Task", "TodoWrite", "NotebookEdit"]
  },
  ...
}
```

- `WebSearch` — 리서치할 URL을 *먼저 검색*
- `WebFetch` — 검색한 URL의 *본문 가져오기*
- `Write` — 새 `<sector>-analyst/SKILL.md` *파일 작성*

### 3) 헤드리스 실행

```bash
cd ~/presentations/20260514/demo/05-meta-skill
claude -p \
  --setting-sources project \
  --dangerously-skip-permissions \
  "통신사 관련 종목의 주식 섹터 전문가 스킬을 생성하자"
```

`**--dangerously-skip-permissions` 가 필수인 이유:**
헤드리스 모드(`-p`)는 사용자 승인을 못 받는다. `Write`가 `allow`에 있어도 `*.claude/` 디렉토리에 쓰는 행위*는 **자가-수정(self-modifying) 작업**이라 추가 확인 프롬프트가 자동 발동돼 *두 번 거부*된다. `--dangerously-skip-permissions` 플래그가 *모든 권한 확인을 건너뛰어* 헤드리스에서 무중단 실행을 보장한다.

> ⚠ **안전 메모:** 이 플래그는 `deny` 목록까지 우회한다. 발표 환경처럼 *제어된 시연 디렉토리*에서만 사용. 실제 운영 환경에서는 신중히.

>
> 잠깐, 한 발 떨어져서 봅시다.
>
> 요즘 *AI를 만드는 AI* 라는 말 들어보셨을 거예요. 더 좋은 AI 모델을 또 다른 AI가 학습시키고, 코드 짜는 AI가 또 다른 AI를 짜내고. 그렇게 **AI 발전 자체가 가속**되고 있다는 얘기. 자기 자신을 더 빠르게 만들어내는 도구가 된 거죠.
>
> 그런 시대에 — *Skill을 만드는 Skill* 은 사실 너무 자연스러운 수순입니다. *AI가 AI를 만드는데, 매뉴얼이 매뉴얼을 만드는 게 어려울 리가 없죠.*
>
> 그리고 이 메타 구조 덕에 — *제가 닿을 수 있는 지식의 경계*와 *생산성*이 같이 올라갑니다. *제가 직접* 100장을 쓰는 대신, *매뉴얼이 매뉴얼을 쓰게* 하는 것. 그래서 — **지식의 경계를 넓히고 생산성을 가속하려면, 메타 도구를 안 쓸 이유가 없어요. 필수입니다.**
>
