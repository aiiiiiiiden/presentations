# 3장 AI의 등장

**시간:** 5:30–10:30 (5분)
> ⚠ 기존 "3:30–8:00 (4.5분)" 추정은 표 4단계 빌드 호흡(5초×4 = 20초)·슬라이드 전환·넷째 단락 부연(~109초)을 누락한 값. 발표자 노트 자체 추정도 4:40으로 이미 10초 초과였음. 시작 시각은 01-intro·02 시프트(+2.0분)에 따라 3:30 → 5:30으로 조정.

**핵심 메시지:** 자기 모순(Don't ↔ Boil)의 *원인*은 시간순으로 일어난 4단계 사실. 의견이 아니라 사실.
**기능:** 청중에게 "16년 사이에 진짜 뭔가 바뀌었구나"를 사실 기반으로 못 박는다.

## 발표 스크립트

> 답을 찾으려면, 지난 4년 사이에 무슨 일이 있었는지 4단계로 짧게 짚어 보겠습니다.
>
> **첫째, 2022년. ChatGPT의 등장**  
> 2022년 하반기에 ChatGPT가 공개되며 누구나 "자연어를 이해하고 답하는 모델"을 손쉽게 써 봤습니다.  
> 처음에는 신기해했지만 좁은 컨텍스트, 낮은 생성 퀄리티, 느린 응답, 잦은 할루시네이션이 겹치며 금세 시들해졌습니다.
>
> **둘째, 2023년. LLM 모델과 생태계의 발전**  
> GPT-3.5 기반의 ChatGPT에 실망의 목소리가 커지던 상황에서 2023년 상반기에 GPT-4가 등장하며 추론 품질이 한 단계 점프했습니다.  
> 지금 모델과 비교하면 GPT-4도 컨텍스트가 좁고 생성 퀄리티가 낮고 응답이 느리고 할루시네이션이 잦았지만, 당시만 해도 "단순한 답변"을 넘어 "여러 곳에서 활용해볼 만하다"는 평가가 많아졌습니다.  
> 특히 RAG, LangChain, VectorDB 같은 LLM 기반 생태계 기술이 쏟아져 나오며, 모델만으로는 넘기 어려운 부분을 생태계로 메우려는 시도가 본격적으로 시작됐습니다.
>
> **셋째, 2024년. GPU 전쟁과 Cursor의 등장**  
> 돌아보면 2024년이 분기점이었습니다.  
> 2024년 1월까지만 해도 50불이던 엔비디아 주가가 가파르게 치솟기 시작했습니다.
>
> 그 시기 IDE 시장에서도 흥미로운 일이 있었습니다.  
> **Cursor**가 소수 인력으로 제품을 빠르게 런칭하고 AI 보조 기능을 앞세워 — *구독료만으로 인프라 비용을 감당한다*는 가능성을 처음 보여줬습니다.  
> 당시 체감한 생산성 향상은 1.5배 정도였지만, 그것만으로도 충분히 만족스러웠습니다.
>
> **넷째, 2025년, Claude Code의 등장과 Agent로의 진화**  
> Claude Code가 정식 출시된 지 이제 1년이 지났습니다.  
> 출시 당시 Claude Code는 자연어 요청을 Cursor보다 더 잘 처리하는 도구였습니다.  
> Cursor 대비 요청을 처리할 계획을 더 촘촘히 세웠고, 결과물도 더 좋았습니다.  
> 어찌 보면 Cursor는 범용 GPT 모델에 코드베이스를 인덱싱하고 시스템 프롬프트만으로 복잡한 요청을 풀어야 했으니 한계가 있었던 셈입니다. 그 한계를 넘어선 Claude Code에 개발자들이 놀랐고, Cursor 구독을 끊고 Claude Code로 갈아탄 사람이 많아졌습니다.
>
> 특히 Command를 시작으로 MCP, Skills, Sub-Agent까지 에이전트 개발 환경의 표준을 먼저 제시하며 단기간에 사용자를 끌어모았습니다. 이때를 기점으로 AI는 보조 도구에서 동료로 인식이 바뀝니다.
>
> 2024년이 GPU의 시대였다면, 2025년은 모델의 입출력 컨텍스트가 폭발하면서 GPU와 함께 메모리의 중요성이 부각된 해입니다.  
> 1년 전 SK하이닉스 주가가 20만원이었으니, 주식 시장은 이 AI의 흐름을 꽤나 정직하게 반영하고 있던 셈입니다.
>
> 이 4단계가 **3년 반** 동안의 이야기입니다.
>
> Garry Tan이 2010년에 "Don't boil the ocean"이라 말했을 때, 이 4단계 중 *어느 것도 일어나기 전이었습니다.*  
> 바다를 끓이지 말고 핵심에 집중하라던 Garry Tan이 16년이 지난 지금은 바다를 끓이라 말합니다.  
> Garry Tan의 가치관이 바뀐 게 아니라, 세상이, 기술이 바뀐 겁니다.

## 발표자 노트

- **시각 자료:** 같은 표에 한 줄씩 단계적으로 추가되는 빌드(4번 클릭). 한 줄 노출 후 5초씩 호흡.
- **호흡 (5:00 기준):** 인트로 8초 / 첫째 30초 / 둘째 60초 / 셋째 47초 / 넷째 109초 / 마무리 42초 ≈ 4:56 순수 발화. + 표 빌드 호흡 5초×4 (20초) + 슬라이드 전환·시선 (10초) ≈ **5:25**. 15~25초 여유라 안정 페이스로 진행 가능. **단축 옵션** — 4.5분으로 줄이려면 빠른 페이스(320 spm) + 둘째 마지막 단락(RAG/LangChain/VectorDB 부연) 한 호흡 압축. **확장 옵션** — 5.5분으로 늘리려면 넷째의 *"AI 보조 도구 → AI 동료"* 한 줄에 3초 침묵 추가.
- **시선:** "**3년 반**"은 손으로 숫자를 그리며 강조. 16년 vs 3년 반의 대비가 핵심.
- **금기 표현:**
  - "AI 혁명", "패러다임 시프트" 같은 추상어 금지 — 사실 4개로만 말함.
  - 특정 회사를 비판하거나 칭찬하는 표현 금지 (OpenAI/Anthropic/Cursor/Claude Code 모두 사실로만).
- **사실 정확성:** 셋째·넷째 단락에 *검증 가능한 수치/주장*이 다수(엔비디아 50불, 1.5배 생산성, SK하이닉스 20만원, Claude Code 정식 출시 1년). 발표 직전 한 번 더 확인.
- **마무리 톤:** "Garry Tan의 가치관이 바뀐 게 아닌, 세상이, 기술이 바뀌었다" — 이 한 줄이 5장 *"둘 다 맞는 말이었습니다"* 회수의 *논리적 기반*을 제공한다. 청중 머릿속에 *원인은 외부(세상/기술)*라는 인식을 심고 5장으로 넘긴다.
- **다리 문장:** "이건 Garry Tan의 가치관이 바뀐게 아닌, 세상이, 기술이 바뀌었다는 걸 말해주고 있습니다." → 0.5초 정지 → "그래서 이 변화가 가져온 7개 단어를 — 짧게 정리하고 데모로 넘어가겠습니다." (4장 진입)
