# 6-7 데모 7단계 — Agent (터미널 밖에서 자율적으로 일하는 AI)

**시간:** 35:00–38:00 (3분)

> ⚠ 시작 시각은 01~06-6 시프트(+8.5분) 누적에 따라 결정. 06-6이 35:00에 끝나므로 06-7 시작 35:00.

**시연 개념:** **Agent** — *내가 트리거하지 않아도* 일하는 AI. 메시지로 부탁받고(즉시 또는 *예약*), 정해진 시간에 스스로 시작하고, 상태를 지켜보다 조건 만족 시 알아서 움직인다.
**한 줄 요약:** 지금까지는 *터미널에서 내가 부르면* AI가 응답. 이번엔 *메시지·시간·상태* 셋 다 트리거. 같은 도구가 *자율적으로 일하는 동료*가 된다.
**연출 형식:** **라이브 데모** — 발표자의 휴대폰 화면을 송출, Telegram 대화 + 알람 + 자동 발송 결과를 그대로 노출.
**청중 반응 목표:** "어, 이건 그냥 도구가 아닌데? 비서를 한 명 두는 느낌이네."

### 1) "지금까지"의 한계 짚기

6-1~6-6에서 만든 AI는 모두 *내가 터미널에서 부르면 응답*하는 패턴.

| 무엇이 좋아졌나 | 무엇이 아직 부족한가 |
|---|---|
| 분석 깊이 (페르소나·스킬) | *내가 매번 명령*해야 시작 |
| 매뉴얼 자동 생성 (메타) | *내가 터미널 앞에 앉아 있어야* 함 |
| 외부 지식 통로 (MCP) | *밤·주말·휴가 중*엔 멈춰 있음 |

→ 압도적인 생산성을 내기엔 **사람이 병목**.

### 2) Agent 정의 되새기기 (4장 회수)

> **Agent = 목표가 주어지면 *스스로 도구를 골라*, *스스로 실행 시점을 정해*, *스스로 결과를 평가해 다음 행동을 결정*하는 AI.**

지금까지는 *도구를 고르는* 절반만 본 셈. 이번 단계에서 *나머지 절반(자율 실행·자율 트리거)*을 채운다.

트리거를 *세 종류*로 나눠 시연:

| 트리거 | 데모 | 명령어 |
|---|---|---|
| **메시지** | Telegram 즉시 + *예약 알람* | (휴대폰에서 봇한테 메시지) |
| **시간** | 매일 정해진 시각 자동 실행 | `crontab` / `launchctl` |
| **상태** | 시장 조건 만족 시 자율 실행 | `npm run monitor` |

### 3) 라이브 데모 ① — 메시지 트리거 (즉시 + 예약)

#### 3-a. 즉시 분석

발표자의 휴대폰을 송출. 텔레그램에서 봇한테:

```
SKT 종목 분석 리포트 작성해줘
```

봇 응답 (5~15초):
```
🛠 KIS 매뉴얼 조회: get_current_price, get_financials, get_chart
## 📊 핵심 데이터 (출처: KIS Open API)
| 항목 | 값 |
| 현재가 | 55,300원 |
| PER / PBR | 9.8 / 0.92 |
...
```

→ **터미널 한 번도 안 열고** 휴대폰만으로 *실 숫자가 박힌* 리포트 도착.

#### 3-b. 예약 알람 (지연 메시지 트리거)

이어서 같은 채팅에 자연어로:

```
30초 뒤 SKT 단신 정리해줘
```

봇 즉시 응답:
```
✅ 알람 등록
abc12345
⏰ 26/5/13 15:30:00 (30초 뒤)
📝 SKT 단신 정리해줘
취소: 알람 삭제 abc12345 또는 /cancel abc12345
```

알람 목록 확인:

```
알람 목록
```

→ 등록된 알람 일람.

30초 후 — *발표자가 다른 슬라이드로 넘어가는 사이* — 봇이 자율적으로:
```
⏰ 예약 알람 — 30초 뒤
작업: SKT 단신 정리해줘

(분석 본문)
```

알람은 `data/alarms.json` 에 저장되므로 **봇을 껐다 켜도 살아남는다** (catch-up 발사).

### 4) 라이브 데모 ② — 시간 트리거 (스케줄러)

```bash
# crontab 한 줄 (사전 등록, 노출만)
0 7 * * * /Users/cody/presentations/20260514/demo/07-agent/scheduler/daily.sh
```

발표자 휴대폰의 *오늘 아침 7시 자동 발송 메시지*를 스크롤로 노출:

```
📅 자동 리포트 — 2026-05-13 07:00 (KST)
🛠 KIS 매뉴얼 조회: get_current_price, get_financials
(SKT 분석 본문)
```

→ *내가 잘 때도* 일이 진행된다.

### 5) 라이브 데모 ③ — 상태 트리거 (모니터)

`src/monitor.ts` 가 백그라운드에서 KOSPI 변화율을 폴링 (시뮬레이션, 60초 tick). `MONITOR_THRESHOLD_PCT=-3` 만족 시 자동 트리거.

발표 중 다른 터미널에서 *수동 트리거*:

```bash
npm run monitor:fire
```

→ `monitor.trigger` 파일이 생기고, 다음 폴 사이클에 모니터가 발견 → 즉시 트리거. 휴대폰에:

```
🚨 시장 경보 — KOSPI 급락 감지 (수동 트리거)
시각: 2026-05-13 15:32:14 (KST)
지수: 2475.32 (-3.27%)

통신 3사 점검을 위한 분석 매뉴얼을 한투 자료실에서 가져옵니다...
```

5~15초 뒤 분석 결과 메시지 도착.

→ *내가 모르는 사이*에도 위험을 잡아온다.

## 발표 스크립트

> 일곱 번째 단계 — **Agent**입니다.
>
> 자, 지금까지 6단계 함께 보셨는데 — 한 발 떨어져서 보면 우리가 만든 건 다 같은 패턴이에요. *제가 터미널에 명령어를 치면, AI가 똑똑하게 응답*하는 패턴. 분석 깊이는 깊어졌고, 매뉴얼은 AI가 직접 쓰고, 외부 자료실까지 닿게 됐죠. 좋아요. 그런데 — *제가 부르지 않으면* 아무 일도 안 일어났습니다.
>
> 압도적인 생산성을 내려면, 이 *제가 트리거해야 움직이는* 구조 자체를 깨야 해요.
>
> 4장에서 *Agent*가 뭐라고 했죠? — *목표를 주면, 스스로 도구를 고르고, 스스로 실행 시점을 정하고, 스스로 결과를 평가해서 다음 행동을 결정하는 AI*. 지금까지는 *도구를 고르는* 쪽만 봤어요. 이번엔 나머지 절반 — *언제 시작할지를 스스로 정하는* 부분을 봅니다.
>
> 트리거를 세 종류로 나눠 보여드릴게요. **메시지·시간·상태**.
>
> ---
>
> **첫째 — 메시지로 부르기.**
>
> 제가 지금 휴대폰을 꺼냅니다.
>
> *(휴대폰 화면 송출 시작)*
>
> 평범한 텔레그램이에요. 이 봇이 — 제 노트북에서 도는 Claude Code랑 연결돼 있습니다. 한 번 말 걸어볼게요.
>
> ```
> SKT 종목 분석 리포트 작성해줘
> ```
>
> 보세요. 응답이 옵니다. 6-6에서 본 *설계서*가 아니라, 이번엔 *진짜 숫자*까지 박힌 리포트예요. 현재가 55,300원, PER 9.8 — 한투 시스템에서 *지금 이 순간* 받은 값입니다. 그런데 — *저는 터미널을 한 번도 안 열었어요*. 휴대폰만으로요. 지하철에서, 화장실에서, 외근 중에 — 일이 시작됩니다.
>
> 한 가지 더 보여드릴게요. 같은 봇한테 *지금 말고, 좀 이따가* 일하라고 시킬 수 있습니다.
>
> ```
> 30초 뒤 SKT 단신 정리해줘
> ```
>
> 알람 등록 확인이 즉시 도착하죠. 약속 ID, 시각, 작업 내용까지 정리해서요. 이 약속을 *파일에 저장해둡니다*. 봇을 껐다 켜도 살아남아요. 등록해두고 잊고 다른 슬라이드로 넘어가도, 30초가 지나면 봇이 알아서 분석 시작하고 결과를 *제 휴대폰으로* 던집니다.
>
> 발표 마지막쯤 한 번 도착할 거예요. 기다려보시죠.
>
> ---
>
> **둘째 — 시간이 부르기.**
>
> *(crontab 한 줄 노출)*
>
> 컴퓨터에 *cron*이라는 게 있어요. *"매일 7시에 이거 실행해"* 같은 약속을 한 줄로 적어두면, 알아서 그 시간에 동작합니다. 거기에 — 방금 본 분석 명령을 박아뒀어요.
>
> *(휴대폰 — 오늘 아침 7시 자동 발송 메시지 스크롤)*
>
> 보세요. 오늘 아침 7시, 제가 자고 있을 때 — *SKT 분석 리포트*가 텔레그램에 들어와 있었습니다. 제가 일어나서 커피 내리는 시간에, 일은 이미 끝나 있었습니다.
>
> ---
>
> **셋째 — 상황이 부르기.**
>
> 마지막 데모. *조건부 자율 실행*입니다.
>
> 에이전트한테 미리 알려뒀어요. *"코스피가 -3% 떨어지면 보유 종목 점검하고 알려줘"*.
>
> *(별도 터미널에서 `npm run monitor:fire` 실행)*
>
> 보세요 — 휴대폰에 *경보*가 떨어졌습니다. 제가 시키지 않았는데. *시장 상태*가 트리거가 됐죠. 에이전트는 즉시 통신 3사 점검 매뉴얼을 한투 자료실에서 가져와서 *판단 근거 한 묶음*을 던져줍니다.
>
> ---
>
> 정리하면 — *Agent*는 *사람이 트리거*하는 도구가 아니에요. **메시지·시간·상태가 트리거**하는 도구입니다. 그래서 *내가 자고 있어도, 외근 중이어도, 휴가를 가도* — 일은 진행됩니다.
>
> 이게 *압도적 생산성*의 본질입니다. *내가 일을 100배 빨리 하는* 게 아니라, *나 없이도 일이 진행되는* 구조. 이걸 만드는 게 — 에이전트입니다.
>
> 그런데 한 가지 — *이 정도 일하는 무언가*를, 우리가 *도구*라고 부를 수 있을까요? 이게 다음 이야기입니다.
>
> *(아 — 마침 휴대폰에 알람 도착. *"30초 뒤 SKT 단신"* 결과가 도착하면서 자연스러운 클로징.)*

## 발표자 노트

- **연출 의도:** 데모 클라이맥스. *"AI = 내가 부르면 응답하는 것"* 이라는 청중의 기본 모델을 *완전히 깨는* 자리. 라이브 텔레그램이 결정타. 알람이 발표 후반에 *자연스럽게 도착*해 클로징을 마무리하면 *짠* 하는 효과.
- **호흡 (3분 분배):**
  - ① 6-1~6-6 한계 + Agent 정의 (4장 회수) + 3종 트리거 예고 — 약 30초
  - ② 데모 ① 즉시 분석 (휴대폰 → KIS 실데이터 리포트) — 약 35초
  - ③ 데모 ① 예약 알람 ("30초 뒤 ..." 등록 + 약속 ID 노출) — 약 25초
  - ④ 데모 ② 스케줄러 (어제 7시 메시지 스크롤) — 약 30초
  - ⑤ 데모 ③ 모니터 (수동 트리거 + 경보 도착) — 약 35초
  - ⑥ 정리 — *"메시지·시간·상태가 트리거"* + 압도적 생산성 정의 — 약 25초
  - ⑦ 07 클로징 다리 + 알람 자연 도착 — 약 10초
  - **총 ~190초 = 3:10** (10초 버퍼, 약간 타이트)
- **시각 자료:**
  - 휴대폰 송출 (가장 중요) — Telegram 대화창. 가능하면 *큰 화면*에 미러링.
  - 알람 등록 확인 메시지 — *ID·시각·작업* 표시 부분 강조 박스.
  - `crontab` 한 줄 — 캡처 한 장.
  - 오늘 아침 7시 메시지 — 휴대폰 알림 센터 스크롤.
  - 모니터 경보 알림 — 잠금화면에 알람 떨어지는 모습.
- **핵심 회수 한 줄:** *"Agent는 사람이 트리거하는 도구가 아니라, 메시지·시간·상태가 트리거하는 도구."*
- **부가 회수 한 줄:** *"압도적 생산성 = 내가 일을 100배 하는 게 아니라, 나 없이도 일이 진행되는 구조."*
- **알람 활용 팁:** *데모 ① 후반에 30초 알람* 등록 → 발표 마지막 06-7→07-conclusion 다리 직전에 자연스럽게 도착하면 *살아 있는 에이전트* 인상이 강해진다. *반드시* 리허설에서 도착 타이밍 측정.
- **금기 표현:**
  - "AI 비서" — 비유로는 OK지만 *과의인화 금지*. *내가 가르친 절차*를 자율 실행한다는 정확한 표현.
  - "AI가 알아서 다 합니다" — 과장 금지. *목표·조건·도구는 사람이 설계*하고, 에이전트는 *그 안에서* 자율.
  - "이제 사람이 필요 없어요" — 절대 X. 청중에게 *해고 공포* 심으면 발표 망함. *"사람은 더 큰 일을 한다"* 톤 유지.
  - "특이점·AGI" — 6-5 줌아웃에서 이미 가드. 여기서도 안 건드림.
- **사전 준비 (가장 까다로움, 별도 체크리스트):**
  - 텔레그램 봇 토큰 + chat ID 화이트리스트 등록 완료.
  - `npm run bot` 백그라운드 가동 + 휴대폰에서 *즉시 분석 + 알람 등록* 양쪽 리허설 1회.
