## 7개 개념 정의 (표)


| 개념          | 1줄 정의                        | 비유                           |
| ----------- | ---------------------------- | ---------------------------- |
| **Model**   | 사전 학습된 추론 시스템 — AI의 두뇌 자체    | 갓 졸업한 신입의 두뇌                 |
| **LLM**     | 자연어로 입력받고 자연어로 답하는 모델        | 그 신입이 한국어를 한다                |
| **Prompt**  | 모델에 1회 전달되는 입력 지시문           | 그날의 발주서 한 줄                  |
| **Skill**   | 영구 저장된 절차 명세 — 반복 적용되는 매뉴얼   | 회사 표준 작업 매뉴얼(.md)            |
| **Agent**   | 목표를 받아 자율 작동하는 상위 모드         | 매뉴얼·발주서·도구를 *알아서* 조합해 일하는 사람 |
| **MCP**     | 외부 데이터·도구 연결의 표준 프로토콜        | 외부 세계로 뚫린 안전한 통로             |
| **Harness** | Agent 실행 환경의 총체 — 권한·훅·설정의 합 | 그 사람이 일하는 사무실(권한·도구·규칙 포함)   |

## 발표 스크립트

> 데모로 들어가기 전에, 오늘 내내 등장할 단어 7개를 짧게 정리하겠습니다.
>
> 요즘 AI 이야기를 하면 빠지지 않는 단어들이죠. Model, LLM, Prompt, Skill, Agent, MCP, Harness.  
> 7개를 3개 그룹으로 묶어보겠습니다.  
> 첫 번째 그룹은 Model, LLM, Prompt.  
> 두 번째는 Skill과 Agent.  
> 세 번째는 MCP와 Harness입니다.
>
> ### 블록 1 — Model, LLM, Prompt (1분)
>
> 첫 번째 그룹부터 봅시다. Model, LLM, Prompt.  
> **Model**은 사전 학습된 추론 시스템, 곧 AI의 *두뇌 자체*입니다.  
> **LLM**은 대량의 텍스트로 학습되어 자연어를 이해하고 생성하는 데 특화된 Model입니다.  
> **Prompt**는 LLM에 건네는 입력 지시문입니다.  
>
> 예를 들어 "SK하이닉스의 주가를 예측해서 알려줘"라고 입력해봅시다.  
> 이 *문장 한 줄*이 바로 **Prompt**입니다.  
> 이 Prompt를 받아 *한국어로 알아듣고 한국어로 답하는 시스템*이 **LLM**, 그 안에서 *실제로 추론을 돌리는 사전 학습된 두뇌*가 **Model**입니다.
>
> ### 블록 2 — Skill, Agent (2분)
>
> 두 번째 그룹, Skill과 Agent.  
> AI에게 역할을 부여하고, 그 역할을 *자율적*으로 굴리는 짝입니다.  
>
> **Skill**은 반복해서 쓰려고 *미리 적어둔 매뉴얼*입니다.  
> **Agent**는 목표를 받아 *스스로 판단하고 움직이는 실행 모드*입니다. 맥락에 따라 의미가 조금씩 달라지는 단어죠.  
>
> Claude Code 같은 도구를 띄우면 미리 정의해둔 Skill(매뉴얼)과 Agent(역할)가 함께 로드됩니다.  
> 그러면 Main 에이전트가 사용자의 입력을 기다리다가, 요청이 들어오는 순간 필요한 Skill이나 Agent를 *호출해 일을 처리합니다.*
>
> 다시 "SK하이닉스 주가를 예측해줘"라는 요청이 들어왔다고 해보죠.  
> **Skill**은 "주가 예측은 이런 순서로 한다 — 시세 조회 → 실적 확인 → 섹터 동향 점검 → 리포트 작성"처럼 *작업 절차를 적어둔 매뉴얼*입니다.  
> 이때 그 매뉴얼을 들고 *실제로 데이터를 끌어오고, 결과를 엮어 답하는 실행 주체*가 바로 **Agent**입니다.  
>
> 다시 말해, **Skill은 "어떻게 일할지"를 적어둔 매뉴얼**, **Agent는 "그 매뉴얼을 들고 실제로 일하는 주체"**입니다.
>
> ### 블록 3 — MCP, Harness (3분)
>
> 세 번째 그룹에 MCP와 Harness를 묶었습니다.  
> 둘 다 에이전트가 돌아가는 *환경*에 관한 단어입니다.
>
> **MCP**는 Model Context Protocol의 약자, Agent를 외부 데이터·도구와 이어주는 *표준 프로토콜*입니다.
>
> "SK하이닉스 주가를 예측해줘"라는 요청이 들어왔다고 합시다. Skill로 절차를 아무리 잘 짜놨더라도, *실제 시세, 최근 실적, 외국인·기관 수급, 업황 뉴스* 같은 외부 데이터가 없으면 한 줄도 못 씁니다.
>
> LLM이 가진 지식은 *학습이 끝난 시점*에 멈춰 있습니다. Model 머릿속에는 작년·재작년 데이터까지만 있을 뿐, 지금 이 순간의 종가가 얼마인지, 어제 어떤 실적 발표가 났는지는 모릅니다. Skill에 절차가 아무리 정교해도, 데이터를 가져올 손이 없으면 매뉴얼은 종이 쪼가리죠.  
> 그래서 Agent에게는 증권 API, 공시 사이트, 뉴스 데이터베이스 같은 *바깥 세계*로 뚫린 **통로**가 필요합니다. 그 *표준 통로*가 바로 **MCP**입니다. 증권사 시세 서버든, 환율 API든, 사내 리서치 위키든, 종류는 달라도 MCP라는 *공통 규격*으로 묶여 있어서, Agent는 늘 같은 방식으로 어떤 외부 데이터에도 닿습니다. 바깥과 연결되는 만큼, *매 호출마다 사용자 권한을 받는 안전장치*도 함께 걸려 있습니다.
>
> **Harness**는 Agent가 돌아가는 환경 전체입니다. Model을 뺀 나머지 모든 것, 즉 Prompt, Skill, Agent, MCP, 권한·설정, 입출력까지 묶어 *Agent가 일할 판을 깔아주는 시스템*이죠. Claude Code가 Opus·Sonnet·Haiku 같은 Model을 깔고 Prompt, Skill, Agent, MCP를 동원해 사용자의 요청을 처리하는 그 실행 환경이 Harness입니다.
>
> 자, 이 7개 단어를 머릿속에 띄워둔 채로, 이제 제 *개인 문제* 이야기를 잠깐 하고 데모로 넘어가겠습니다.
