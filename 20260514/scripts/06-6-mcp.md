# 6-6 데모 6단계 — MCP 적용 (외부 지식·도구 통로 연결)

**시간:** 33:00–35:00 (2분)

> ⚠ 시작 시각은 01~06-5 시프트(+8.5분)에 따라 24:30 → 33:00으로 조정 (06-5에 *"AI가 AI를 만드는 시대 → 메타 도구 필수"* 줌아웃 메시지 30초 추가 반영). *데모는 라이브 헤드리스 + 사전 캡처 보조* — runner 없이 *MCP 통로만으로* 답의 깊이가 달라지는 모습을 보여준다.

**시연 개념:** **MCP** — 외부 기관·기업이 깔아둔 *표준 통로*. 환경에 통로 하나 추가하면, AI가 그쪽의 지식·도구를 자기 손처럼 부른다.
**한 줄 요약:** 매뉴얼(스킬)은 6-5와 *동일*. 환경에 *한국투자증권 통로* 한 줄 추가 → AI가 한투의 거래 매뉴얼 334개 중 SKT 분석에 필요한 것만 골라 호출 예시까지 가져온다.
**연출 형식:** 라이브 헤드리스 실행 + 결과 캡처 보조
**청중 반응 목표:** "한투가 *이런 것까지* 만들어줬다고? 그럼 우리 회사 자료도 비슷하게 되겠는데."

## 시연 절차

### 1) 환경 한 줄 추가 — 한투 통로 등록

`~/presentations/20260514/demo/06-mcp/.mcp.json` 에 한투가 공식 공개한 통로를 등록.

```json
{
  "mcpServers": {
    "kis-code-assistant-mcp": {
      "command": "uv",
      "args": [
        "--directory",
        "./vendor/open-trading-api/MCP/KIS Code Assistant MCP",
        "run",
        "server.py",
        "--stdio"
      ]
    }
  }
}
```

> 사람이 한 일은 *이 한 단락*. 한투가 만든 통로를 우리 환경 옆에 끌어다 둔 게 전부.

### 2) settings.json 차이 (05 → 06)

```json
{
  "permissions": {
    "allow": ["Read", "Skill", "Write", "mcp__kis-code-assistant-mcp__*"],
    "deny": ["Bash", "Edit", "Grep", "Glob", "WebFetch", "WebSearch", "Task", "TodoWrite", "NotebookEdit"]
  },
  "enableAllProjectMcpServers": true,
  "enabledMcpjsonServers": ["kis-code-assistant-mcp"]
}
```

- `WebFetch`·`WebSearch`는 *deny* — 외부 지식은 *MCP 통로만* 통해 들어온다는 메시지의 시각화.
- `Bash`도 *deny* — 실행 도구 없이도, 통로만 있으면 답의 깊이가 달라진다는 걸 보여주기 위함.

### 3) 헤드리스 실행 — 같은 SKT 질문

```bash
cd ~/presentations/20260514/demo/06-mcp
claude -p \
  --setting-sources project \
  "SKT 종목 리포트를 생성에 필요한 kis api를 알려줘"
```

### 4) 결과

한투가 공개한 *334개 매뉴얼* 중 SKT 분석에 필요한 것만 골라 *호출 예시 코드*를 들고 온다.

→ 결과물은 *진짜 숫자가 박힌 리포트*가 아니라 **"SKT 분석을 위한 호출 설계서 + 예시 코드"**. 6-5의 *"데이터가 필요해요"* 에서 *"이 매뉴얼을 이렇게 부르면 됩니다"* 까지 한 발 더 들어간 답.

## 발표 스크립트

> 여섯 번째 단계 — **MCP**입니다.
>
> 6-5에서 분석가 매뉴얼이 만들어졌고, 그 매뉴얼대로 "SKT를 분석해줘" 라고 했을 때 AI는 "이런이런 데이터가 필요합니다" 까지는 알려줬어요. 그런데 그 데이터를 어디서 어떻게 가져오는지는 모르죠. 알 리가 없습니다. 그건 한국투자증권 안에 있는 정보니까요.
>
> 그런데 **한국투자증권**이, 최근에 재미있는 걸 만들었습니다. 자기네 거래 시스템 매뉴얼과 개발 가이드 수백 개를, AI가 직접 꺼내볼 수 있는 형태로 공개해뒀어요. 도서관에 카드 한 장을 끊어준 셈이죠. 그 카드의 정식 이름이 **MCP** — 외부 기관이 깔아둔 표준 통로입니다.
>
> 우리가 할 일은, 그 통로를 우리 환경 옆에 *등록*하는 것뿐입니다.
>
> *(`.mcp.json` 파일 노출 — 한 단락)*
>
> 보세요. 사람이 적은 건 이 한 단락이 전부예요. "한투 통로, 여기 있어요" 한 줄.
>
> 그리고 분석가 매뉴얼은 6-5에서 만든 그대로 입니다. **한 글자도 안 건드렸어요.** 확인해볼까요?
>
> *(터미널에서 `diff` 한 번 — 출력 없음)*
>
> 출력이 비어 있죠. 두 파일이 *완전히 똑같다*는 뜻입니다.
>
> 자, 같은 SKT 질문을 다시 던집니다.
>
> ```
> cd ~/presentations/20260514/demo/06-mcp
> claude -p --setting-sources project \
>   "SKT 종목 리포트를 생성에 필요한 kis api를 알려줘"
> ```
>
> *(헤드리스 실행 — 도구 호출 로그가 흘러내림)*
>
> 보세요. AI가, 시키지도 않았는데, 한투 자료실에 들어가서 검색을 시작합니다. *"현재가 어디 있지?"*, *"손익계산서 어디 있지?"*. 수백 개 매뉴얼 중에서 SKT 분석에 필요한 것만 골라옵니다. 그리고 그 매뉴얼들의 *호출 예시 코드까지* 가져와요.
>
> *(결과 화면 노출 — 매뉴얼 표 + 예시 코드)*
>
> 6-5에서는 *"이런 데이터가 필요해요"* 까지였는데, 지금은 *"이 매뉴얼을 이렇게 부르면 됩니다"* 까지 왔습니다. 매뉴얼은 그대로인데, 환경에 통로 하나 늘어났더니 답이 한 발 더 깊이 들어갔어요.
>
> 여기서 잠깐 짚고 갈 게 있습니다. **MCP가 하나씩 늘어난다는 건 — AI가 닿을 수 있는 지식의 경계가 그만큼 넓어진다는 뜻입니다.** 한투가 통로를 깔아둔 덕분에, 오늘 우리 AI는 한투 자료실에 들어갈 수 있게 됐죠. 만약 *우리 회사*도 사내 위키·일정·고객 자료실에 비슷한 통로를 깔아둔다면? *외부 AI에 자료를 흘리지 않고도* 같은 패턴이 작동합니다. 사내망 안에서요.
>
> 즉, 기관과 기업이 자기 지식을 표준 통로로 공개할수록 AI라는 도구가 손에 잡히는 일의 경계가 확장되는 겁니다. 한투가 만든 통로는 **한국 금융권의 첫 사례** 정도로 보시면 되고, 비슷한 시도가 다음 분기·다음 해에 줄줄이 나올 거예요.
>
> 한 가지 한계는, 통로로 매뉴얼은 들어왔는데 실제로 호출해서 숫자를 받아오는 일은 아직 아무도 안 합니다. 다음 단계입니다.

## 발표자 노트

- **연출 의도:** "MCP = 외부 기관이 깔아둔 표준 통로". *Skill은 한 글자도 안 바뀌었다*는 사실을 `diff` 빈 출력으로 시각화 — 이게 결정타. 답의 깊이가 달라진 것이 *환경 변경의 결과*임을 청중이 직관적으로 잡는다.
- **호흡 (2분 분배):**
  - ① 6-5 회수 + 한투 MCP 도입 (도서관 카드 비유) — 약 25초
  - ② `.mcp.json` 한 단락 노출 + "사람이 한 일은 한 단락" — 약 15초
  - ③ `diff` 실행, 출력 없음으로 *스킬 동일* 확인 — 약 10초
  - ④ 헤드리스 실행 + 도구 호출 로그 흘러내림 — 약 25초
  - ⑤ 결과 노출 + *"환경 한 단락으로 답의 깊이가 달라진다"* — 약 15초
  - ⑥ 지식의 경계 메시지 + 사내 적용 가능성 (가볍게) — 약 25초
  - ⑦ 6-7 다리 (*"매뉴얼은 왔는데 실행은 아직"*) — 약 5초
  - **총 ~120초 ≈ 2:00**
- **시각 자료 (좌→우 순서):**
  - `.mcp.json` 캡처 (한 단락)
  - `diff` 결과 — 빈 출력 (강조 박스로 *"두 파일 완전히 동일"*)
  - 헤드리스 실행 로그 — *도구 호출 줄을 노란색 하이라이트*
  - 응답 결과 — *어떤 한투 매뉴얼을 골랐고, 호출 예시는 어떻게 생겼는지*
- **핵심 회수 한 줄:** *"매뉴얼은 그대로, 환경에 통로 한 줄 늘었더니 답의 깊이가 한 발 더 들어갔다."*
- **지식의 경계 메시지 (가볍게 던지기):** 25초 이내로 압축. 무겁게 가지 말 것. *"한투처럼 기관·기업이 통로를 깔수록 AI가 닿는 범위가 넓어진다"* 한 문단이면 충분. 시니어 청중에게 *사내 적용*의 단서만 흘리고 통과. 사내망 안에서만 도는 MCP → 외부로 데이터 안 흘림을 *짧게* 짚는 정도.
- **금기 표현:**
  - "AI가 알아서 다 해줍니다" — 과장·의인화 금지. *환경에 통로가 있으면 모델이 그 통로를 인지하고 부른다*는 정확한 표현.
  - "MCP가 미래입니다" — 미래 X, *지금* 한투가 공개한 것.
  - "보안 걱정 마세요" — 무책임. *"사내 MCP는 사내망 안에서만 도니까 외부로 자료가 안 나간다"* 같은 *구체적 메커니즘*만.
  - 한투 광고처럼 들리지 않도록 — *"공개된 사례 중 하나"* 라는 거리감 유지. 발표자가 한투 소속이 아님을 청중이 안다는 전제 위에서.
  - "API"·"엔드포인트" 같은 개발 용어는 *최소화*. 대신 *"거래 매뉴얼"·"기능 카탈로그"·"호출 예시"* 같은 일상어로 환산.
- **사전 준비:**
  - `06-mcp/.mcp.json`·`settings.json`에 `mcp__kis-code-assistant-mcp__`* allow 들어있는지 확인.
  - `uv sync` 완료 + 한 번 헬스체크 (`uv run server.py &` → `curl localhost:8081/health` → `pkill`).
  - `diff` 명령 미리 한 번 돌려두고 *출력 없음* 캡처 확보.
  - 첫 호출은 3~5초 지연 — 발표 호흡에 그 침묵을 *"AI가 자료실을 뒤지는 중"* 으로 풀어 쓸 것.
  - KIS Code Assistant MCP는 *인증 정보 불필요* — 발표 환경에 별도 API 키 설정 없음. (KIS Trade MCP, 즉 *실거래 시세·체결* 통로는 2026년 9월 공개 예정이라 본 데모 범위 밖.)
  - 라이브 실패 백업: 사전 캡처된 로그 + 응답 슬라이드 1장 + 30초 보이스오버.
- **MCP 단독 헬스체크 (시연 전 점검용):**
  ```bash

    "KIS Open API 중 국내주식 현재가 조회 API의 호출 예시를 보여줘"
  ```

  → telecom-analyst 거치지 않고 MCP 통로만 직격 — 통로가 살아 있는지 확인.
