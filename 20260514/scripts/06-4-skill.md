# 반도체 섹터 전문가

당신은 한국 반도체 산업의 베테랑 애널리스트입니다.

1. 메모리 사이클 — DRAM·NAND ASP·B/G, HBM 양산 캐파, 빅테크 캐펙스
2. 재무 정합성 — P/B, ROE, 영업이익률, FCF (사이클 평균 대비)
3. 공급 측 — SK하이닉스·삼성전자·마이크론 로드맵, 후공정, LTA 비중

5. 리스크 — 사이클 전환, 캐펙스 둔화, 환율, LTA 변화

## 발표자 노트

- **시각 자료:** SKILL.md 파일 내용을 *모니터에 크게* 띄운다. 청중이 "이게 한 장이라고?" 직접 확인할 수 있게. 본문은 30줄을 넘기지 말 것 — 발췌해 띄움. YAML 프론트매터(`name`·`description`)는 강조 표시.
- **연출 의도:** *"코드가 아니라 한국어"* 라는 메시지를 시각으로 못 박는다. 비개발자 청중에게 결정적인 순간. 4장 *Skill 개념*(매뉴얼) 회수도 자동.
- **6-3과의 좌우 비교 슬라이드:** 가능하면 *"6-3: 페르소나 한 줄이 들어간 긴 프롬프트"* vs *"6-4: 페르소나 없는 짧은 프롬프트"* 를 나란히. 결과는 *동일*하다는 사실을 시각으로.
- **호흡 (2분 분배 / 라이브 실행 없음):**
  - ① 도입 + Skill 저장 개념 ("매번 다시 쓰는 비효율") — 약 20초
  - ② SKILL.md 노출 + 설명 ("코드 아니고 한국어") — 약 30초
  - ③ 명령 박스 표시 + 사전 캡처 응답 화면 flash — 약 15초
  - ④ 응답 비교 3 bullets + 에이전트 다운 면모 한 단락 — 약 50초
  - ⑤ 6-5 다리 ("매번 한땀 한땀 적어야 할까?") — 약 5초
  - **총 ~120초 ≈ 2:00**
- **결과 화면 운용:** 라이브 실행 없이 *사전 캡처된 응답 슬라이드*를 띄움. 청중이 표를 다 읽을 시간은 주지 말고, *"이런 결과가 나옵니다"* 한 마디 후 곧장 3 bullet 비교로 이동. 응답 표는 좌우 분할로 6-3 응답과 동시 노출 — *깊이·구조·안전장치 차이*를 시각으로 보여줌.
- **자동 트리거 vs 명시 호출:** 기본은 *자동 트리거*(키워드 매칭). 리허설에서 안 잡히면 프롬프트 맨 앞에 `/semiconductor-analyst` 슬래시 명령으로 명시 호출. 발표 환경의 안전성 우선.
- **금기 표현:**
